I denna inlämning så har jag använt mig mestadels utav att felsöka genom att ändra attribut i html dokumentet eller css dokumentet
och sen spara för att se hur det ser ut på hemsidan. Det var först när hemsidan var klar innehållsmässigt och css var i fokuset som
jag laddade ned firefox och började använda inspekt fönstret för att felsöka problemen som uppstod med responsiviteten jag hade.

Det jag gillade med firefox inspect var att man på ett enkelt sätt kan se vad det är som möjligtvis sätter "käppar i hjulet" på sin css.
Skillnaden mellan att felsöka i chrome i jämförelse med firefox är just inspect valmöjligheten i firefox, och att den känns mer anpassad för just detta.
Men för mig i detta skede som nybörjare så känns en kombination fortfarande optimalt då man börjar utanför firefox och sen kommer till det när sidan tagit form.
